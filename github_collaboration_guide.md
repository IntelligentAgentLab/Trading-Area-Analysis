# 📢 [공지] 스터디 깃허브 협업 가이드: PR(Pull Request) 활용 방법

우리 스터디는 단순한 파일 공유를 넘어, 실무 개발 조직에서 사용하는 **'PR(Pull Request) 기반의 협업 방식'**을 채택했습니다. 처음엔 브랜치를 만들고 PR을 올리는 과정이 번거롭게 느껴질 수 있지만, 이 방식을 통해 코드 충돌을 막고 서로의 자료에 대해 가장 깊이 있게 토의할 수 있습니다.

성공적인 스터디 진행을 위해 아래의 4단계 상세 프로세스를 꼭 확인해 주세요!

## 단계 1: 나만의 작업 공간 만들기 (Branch 생성)
자료를 올릴 때는 절대 `main` 브랜치에 직접 파일을 업로드하지 않습니다. 반드시 내 이름이 들어간 '새 브랜치'를 만들고 시작합니다.

* **상세 방법:** 
    1. 깃허브 레포지토리 좌측 상단의 `main` 버튼을 클릭합니다.
    2. 입력창에 `week1-jiwoo` (예: week1-이름) 형식으로 적고 **Create branch**를 누릅니다.
* **이 과정이 왜 필요한가요?**
    * 작업 공간을 완벽히 분리하기 위해서입니다. `main` 브랜치 하나만 사용하면 다른 사람의 파일과 덮어씌워지거나 꼬이는 문제가 발생할 수 있습니다. 브랜치를 분리하면 내 작업물에만 온전히 집중할 수 있습니다.

## 단계 2: 자료 업로드 및 토의 요청하기 (PR 생성)
내 브랜치에 파일을 올렸다면, 이제 팀원들에게 "내 자료를 메인에 합쳐도 될지 검토해 줘!"라고 요청할 차례입니다.

* **상세 방법:**
    1. 내 브랜치(`week1-이름`)가 선택된 상태에서 주차별 폴더(예: `Week_01/`)에 코드나 PDF 등을 업로드(Commit)합니다.
    2. 업로드 직후 레포지토리 상단에 뜨는 노란색 창의 **[Compare & pull request]** 버튼을 누릅니다.
    3. 제목을 `[1주차] OOO 자료 제출`로 적고, 본문에 이번 주 작업의 핵심 요약과 팀원들에게 묻고 싶은 질문을 적은 뒤 **Create pull request**를 누릅니다.
* **이 과정이 왜 필요한가요?**
    * PR은 일종의 '공식 결재 서류'이자 '토론의 장'을 여는 버튼입니다. PR 본문에 고민되는 지점을 미리 적어두면, 리뷰어들이 어떤 부분에 집중해서 피드백을 줘야 할지 명확한 가이드를 받을 수 있습니다.

## 단계 3: 서로의 자료 뜯어보기 (Review & Comment)
스터디 모임 전까지, 팀원들이 올린 PR에 들어가 자료를 읽고 피드백을 남깁니다.

* **상세 방법:**
    * **코드(.py)나 마크다운(.md) 파일:** PR 화면의 **[Files changed]** 탭에서 코드를 줄 단위로 확인합니다. 특정 줄 번호 옆의 `+` 버튼을 눌러 피드백이나 질문을 남깁니다.
    * **PDF, 워드, 엑셀 파일:** 파일을 다운로드해 확인한 뒤, PR 메인 화면의 대화창(Conversation) 맨 아래에 전반적인 감상이나 질문을 남깁니다.
* **이 과정이 왜 필요한가요?**
    * 대면 회의 시간을 획기적으로 아껴줍니다. 미리 코드를 읽어보고 댓글로 1차 피드백을 주고받으면, 오프라인(또는 화상) 모임에서는 깊이 있는 문제 해결 토의로 바로 넘어갈 수 있습니다.

## 단계 4: 회의록 작성 및 최종 아카이빙 (Merge)
오프라인/화상 회의가 끝나면 해당 주차를 깔끔하게 마무리합니다.

* **상세 방법:**
    1. 주차별 담당 서기가 팀원들이 나눈 피드백 결과를 요약해 `회의록.md` 파일을 작성하고 올립니다.
    2. 각자의 자료에 문제가 없다면, PR 화면 하단의 초록색 **[Merge pull request]** 버튼을 눌러 `main` 브랜치에 합칩니다.
    3. 다 쓴 개인 브랜치는 **[Delete branch]**를 눌러 깨끗하게 지워줍니다.
* **이 과정이 왜 필요한가요?**
    * 파편화된 자료를 하나의 완성된 스터디 결과물(`main`)로 모으는 최종 단계입니다. 이 과정을 거쳐야 나중에 누군가 레포지토리에 들어왔을 때, 매주 어떤 고민을 했는지 한눈에 파악할 수 있는 훌륭한 포트폴리오가 완성됩니다.
